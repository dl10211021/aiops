# 网络交换机巡检清单

本文档提供标准的交换机巡检项目清单和判断标准。

## 巡检级别

### 1. 基础巡检（每日/自动化）
快速健康检查，用于日常监控。

### 2. 标准巡检（每周/每月）
全面的设备状态检查，用于定期维护。

### 3. 详细巡检（按需/故障排查）
深度检查和故障诊断，用于问题定位。

---

## 基础巡检项目

### 1.1 系统运行状态
- [巡检项目] 系统版本信息
- [检查命令] H3C: `display version` / 华为: `display version`
- [检查内容]
  - 设备型号
  - 软件版本
  - 运行时间
  - 设备序列号
- [判断标准]
  - 运行时间过短（<1小时）可能表示意外重启
  - 记录当前版本用于后续对比

### 1.2 CPU使用率
- [巡检项目] CPU负载
- [检查命令] H3C: `display cpu-usage` / 华为: `display cpu-usage`
- [检查内容]
  - 当前CPU使用率
  - 5秒/1分钟/5分钟平均值
- [判断标准]
  - ✅ 正常：< 50%
  - ⚠️ 警告：50-80%
  - ❌ 严重：> 80%
- [处理建议]
  - 80%以上持续超过5分钟需要关注
  - 检查是否有异常流量或攻击
  - 考虑硬件升级或负载均衡

### 1.3 内存使用率
- [巡检项目] 内存占用
- [检查命令] H3C: `display memory` / 华为: `display memory-usage`
- [检查内容]
  - 总内存
  - 已用内存
  - 空闲内存
  - 使用百分比
- [判断标准]
  - ✅ 正常：< 70%
  - ⚠️ 警告：70-85%
  - ❌ 严重：> 85%
- [处理建议]
  - 内存不足可能导致功能异常
  - 检查内存泄漏
  - 考虑升级内存或优化配置

### 1.4 接口状态
- [巡检项目] 接口物理和协议状态
- [检查命令] H3C: `display interface brief` / 华为: `display interface brief`
- [检查内容]
  - 物理状态（UP/DOWN）
  - 协议状态（UP/DOWN）
  - 接口速率
  - 双工模式
- [判断标准]
  - 核心接口应该为UP状态
  - DOWN接口数量异常增加需要关注
  - 记录所有DOWN接口并与上次对比
- [处理建议]
  - 检查线缆连接
  - 检查对端设备状态
  - 查看接口日志

### 1.5 温度状态
- [巡检项目] 设备温度
- [检查命令] H3C: `display environment` / 华为: `display temperature`
- [检查内容]
  - 各部件温度
  - 温度阈值
  - 温度告警
- [判断标准]
  - ✅ 正常：< 50°C
  - ⚠️ 警告：50-65°C
  - ❌ 严重：> 65°C
- [处理建议]
  - 检查机房空调
  - 检查风扇状态
  - 清理灰尘
  - 检查通风

### 1.6 电源状态
- [巡检项目] 电源模块
- [检查命令] H3C: `display power` / 华为: `display power`
- [检查内容]
  - 电源模块状态
  - 电源数量
  - 输入电压
  - 功率
- [判断标准]
  - 所有电源模块应为Normal状态
  - 冗余电源应全部正常
- [处理建议]
  - 电源异常立即更换
  - 检查供电线路
  - 确保冗余

### 1.7 风扇状态
- [巡检项目] 风扇运行
- [检查命令] H3C: `display fan` / 华为: `display fan`
- [检查内容]
  - 风扇状态
  - 转速（RPM）
- [判断标准]
  - 所有风扇应为Normal状态
  - 转速应在合理范围内
- [处理建议]
  - 风扇异常立即更换
  - 避免单风扇运行
  - 定期清洁

### 1.8 告警信息
- [巡检项目] 系统告警
- [检查命令] 华为: `display alarm all`
- [检查内容]
  - 当前活动告警
  - 告警级别
  - 告警类型
- [判断标准]
  - 无Critical或Major告警
  - Minor告警数量可控
- [处理建议]
  - 根据告警类型采取相应措施
  - 记录告警趋势

## 标准巡检项目

包含所有基础巡检项目，以及：

### 2.1 VLAN配置
- [巡检项目] VLAN设置
- [检查命令] H3C/华为: `display vlan`
- [检查内容]
  - VLAN列表
  - VLAN成员接口
  - VLAN状态
- [判断标准]
  - VLAN配置与规划一致
  - 无未使用的VLAN
- [处理建议]
  - 清理无用VLAN
  - 更新VLAN文档

### 2.2 MAC地址表
- [巡检项目] MAC地址学习
- [检查命令] H3C/华为: `display mac-address count`
- [检查内容]
  - MAC地址总数
  - 动态MAC数量
  - 静态MAC数量
- [判断标准]
  - MAC数量在正常范围
  - 无异常MAC地址暴增
- [处理建议]
  - MAC地址暴增可能表示攻击或环路
  - 检查MAC地址表容量

### 2.3 ARP表
- [巡检项目] ARP缓存
- [检查命令] H3C/华为: `display arp` 或 `display arp statistics`
- [检查内容]
  - ARP条目数量
  - 动态/静态ARP
  - ARP老化时间
- [判断标准]
  - ARP数量合理
  - 无ARP欺骗迹象
- [处理建议]
  - 异常ARP检查网络攻击
  - 考虑启用DAI（动态ARP检测）

### 2.4 路由表
- [巡检项目] 路由条目
- [检查命令] H3C/华为: `display ip routing-table`
- [检查内容]
  - 路由协议
  - 路由数量
  - 默认路由
  - 关键路由
- [判断标准]
  - 关键路由存在
  - 无路由环路
  - 路由协议收敛
- [处理建议]
  - 检查路由协议配置
  - 验证路由策略

### 2.5 链路聚合
- [巡检项目] Eth-Trunk/聚合链路
- [检查命令] H3C: `display link-aggregation summary` / 华为: `display eth-trunk`
- [检查内容]
  - 聚合组状态
  - 成员接口
  - 活动成员
  - 负载分担模式
- [判断标准]
  - 所有成员接口应活动
  - 负载均衡正常
- [处理建议]
  - 检查非活动成员原因
  - 验证LACP配置

### 2.6 生成树协议
- [巡检项目] STP状态
- [检查命令] H3C/华为: `display stp brief`
- [检查内容]
  - STP模式
  - 根桥信息
  - 端口角色
  - 拓扑变更
- [判断标准]
  - 无频繁拓扑变更
  - 根桥配置正确
  - 端口角色符合预期
- [处理建议]
  - 频繁变更检查环路
  - 优化STP参数

### 2.7 系统日志
- [巡检项目] 日志分析
- [检查命令] H3C/华为: `display logbuffer reverse | include Error|Warning`
- [检查内容]
  - 错误日志
  - 警告日志
  - 接口状态变更
  - 配置变更
- [判断标准]
  - 无Critical错误
  - 警告在合理范围
  - 接口抖动可控
- [处理建议]
  - 分析错误原因
  - 跟踪重复问题
  - 更新日志服务器

### 2.8 接口流量统计
- [巡检项目] 接口流量和错误
- [检查命令] H3C/华为: `display interface [interface-name]`
- [检查内容]
  - 输入输出流量
  - 错误包数量
  - CRC错误
  - 丢包统计
- [判断标准]
  - 错误率 < 0.1%
  - 无CRC错误
  - 无严重丢包
- [处理建议]
  - CRC错误检查线缆
  - 丢包检查负载
  - 记录流量趋势

## 详细巡检项目

包含所有基础和标准巡检项目，以及：

### 3.1 光模块状态
- [巡检项目] 光模块参数
- [检查命令] H3C/华为: `display transceiver interface`
- [检查内容]
  - 光功率（TX/RX）
  - 温度
  - 电压
  - 厂商信息
- [判断标准]
  - TX功率：-10 到 0 dBm
  - RX功率：-15 到 0 dBm
  - 温度 < 70°C
- [处理建议]
  - 光功率异常检查光纤
  - 温度过高检查散热
  - 记录光模块寿命

### 3.2 硬件诊断
- [巡检项目] 硬件状态
- [检查命令] H3C/华为: `display device`
- [检查内容]
  - 板卡状态
  - 子卡状态
  - 硬件序列号
- [判断标准]
  - 所有板卡Normal
  - 无硬件告警
- [处理建议]
  - 硬件异常及时报修
  - 更新硬件清单

### 3.3 堆叠状态
- [巡检项目] 堆叠健康度（如适用）
- [检查命令] 华为: `display stack`
- [检查内容]
  - 主备角色
  - 成员状态
  - 堆叠链路
  - 优先级配置
- [判断标准]
  - 主备角色明确
  - 所有成员在线
  - 堆叠链路正常
- [处理建议]
  - 堆叠分裂立即处理
  - 优化堆叠优先级

### 3.4 配置备份
- [巡检项目] 配置文件
- [检查命令] H3C/华为: `display current-configuration`
- [检查内容]
  - 当前运行配置
  - 保存配置一致性
  - 配置变更记录
- [判断标准]
  - 运行配置已保存
  - 有定期备份
- [处理建议]
  - 保存重要配置
  - 备份到外部服务器
  - 版本控制管理

### 3.5 用户和权限
- [巡检项目] 登录用户
- [检查命令] 华为: `display users`
- [检查内容]
  - 当前登录用户
  - 登录时间
  - 登录方式
  - IP地址
- [判断标准]
  - 无异常登录
  - 权限配置合理
- [处理建议]
  - 清理无用账号
  - 加强密码策略
  - 启用AAA认证

## 巡检报告模板

### 报告头部
- 巡检日期和时间
- 巡检人员
- 巡检级别（基础/标准/详细）
- 设备清单

### 设备信息
- 设备名称
- IP地址
- 设备型号
- 软件版本
- 序列号
- 运行时间

### 巡检结果摘要
- 总体评估（正常/警告/严重）
- 正常项目数
- 警告项目数
- 异常项目数
- 主要问题列表

### 详细巡检记录
按巡检项目逐项记录：
- 检查项目
- 检查结果
- 状态判断（✅ 正常 / ⚠️ 警告 / ❌ 异常）
- 详细说明
- 建议措施

### 趋势分析
- CPU使用率趋势
- 内存使用率趋势
- 温度变化趋势
- 接口错误趋势

### 处理建议
- 立即处理事项
- 计划处理事项
- 长期优化建议

### 附录
- 关键命令输出
- 详细日志
- 配置备份

## 告警阈值参考

| 指标 | 正常 | 警告 | 严重 |
|------|------|------|------|
| CPU使用率 | < 50% | 50-80% | > 80% |
| 内存使用率 | < 70% | 70-85% | > 85% |
| 温度 | < 50°C | 50-65°C | > 65°C |
| 接口错误率 | < 0.01% | 0.01-0.1% | > 0.1% |
| 光功率(RX) | -10 to 0 dBm | -15 to -10 dBm | < -15 dBm |
| 风扇转速 | > 3000 RPM | 2000-3000 RPM | < 2000 RPM |

## 常见问题处理流程

### CPU使用率过高
1. 确认CPU高负载持续时间
2. 检查接口流量是否异常
3. 查看进程占用（如果支持）
4. 检查是否有攻击流量
5. 考虑硬件升级或分流

### 内存使用率过高
1. 检查内存使用趋势
2. 查看MAC/ARP表规模
3. 检查是否有内存泄漏
4. 重启设备（谨慎操作）
5. 升级软件版本或硬件

### 温度过高
1. 检查机房温度
2. 检查风扇状态
3. 清理设备灰尘
4. 检查通风孔
5. 降低设备负载
6. 考虑增加散热设备

### 接口频繁DOWN/UP
1. 检查物理连接
2. 更换线缆
3. 检查对端设备
4. 查看接口日志
5. 检查接口配置
6. 考虑更换模块或板卡

### 光模块告警
1. 检查光功率
2. 清洁光纤接头
3. 检查光纤是否损坏
4. 更换光模块
5. 检查传输距离是否超标
